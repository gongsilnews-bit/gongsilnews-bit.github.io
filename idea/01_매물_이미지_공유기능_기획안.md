# 🚀 향후 개발 기획: 고객 전송용 매물 이미지 추출 기능

## 기획 배경
- **문제점:** 중개사가 고객에게 매물을 전달할 때, 웹사이트 링크(URL)를 전달하면 사이트 내 타 부동산의 매물이 노출되어 손님을 뺏길 우려가 있음. 화면 캡처는 모바일/PC 환경에 따라 번거로움이 큼.
- **해결책:** 웹페이지 내 매물 정보를 깔끔한 '이미지(JPG)' 형태로 자동 생성하여 전달할 수 있는 기능 제공.

## 구현 아이디어

### 1. 전송용 이미지 카드 생성 (UI/UX)
- 기존 매물 상세 페이지 내 **[고객 전송용 캡쳐]** 또는 **[이미지로 공유하기]** 버튼 추가
- 버튼 클릭 시, 화면에 보이지 않는 백그라운드 렌더링 영역(전용 UI)을 활용
  - 포함할 내용: 메인 매물 사진 1~2장, 핵심 요약 정보 (보증금/월세, 층수, 면적 등), 중개사 이름 및 연락처
  - 뺄 내용: 사이트 로고, 검색창, 기타 메뉴, '다른 매물 보기' 등 불필요한 정보 모두 제거

### 2. 기술 스택: `html2canvas` 또는 `dom-to-image`
- 브라우저 단에서 특정 HTML 요소 배열을 그대로 캔버스(Canvas) 이미지로 변환.
- 서버 리소스(비용)를 들이지 않고 사용자 기기 성능만으로 순식간에 이미지 추출 가능.

### 3. 모바일 편의성 극대화 (Web Share API)
- 모바일 환경 접속 시:
  - 이미지 생성 완료 직후 `navigator.share()` API를 호출하여 고객의 스마트폰에 있는 카카오톡, 문자 메시지 앱 호출
  - 한번의 탭으로 이미지를 즉시 전달할 수 있는 동선 구축.

## 대안 / 고도화 아이디어 (폐쇄형 단독 링크)
- 이미지 용량 제약이나 동적 데이터(비디오, 움직이는 지도)를 보여주고 싶을 경우 활용.
- **동작 방식:** `URL?agent=중개사ID` 파라미터를 추가.
- 페이지 진입 시, 해당 파라미터가 있으면 GNB(상단 메뉴바), 푸터, 다른 매물 검색 영역을 전부 숨김 처리(display: none).
- 고객은 매물을 상세히 둘러볼 수 있으면서도 오직 해당 코드를 전송한 중개사에게만 연락 가능.
