# 🚀 공실뉴스 Next.js + Vercel 이전 계획 (0초 로딩 마이그레이션)

## 📌 배경: 왜 넘어가야 하는가?
현재 공실뉴스는 무료 깃허브 기반의 정적 웹 호스팅(CSR 방식)을 사용 중입니다. 
아무리 서버를 최적화해도 접속 순간 **빈 화면(빈 접시)이 뜨고 👉 내 브라우저가 다시 데이터베이스(Supabase)에 기사를 요청해서 렌더링하는 과정**에서 필연적으로 발생하는 **0.3초의 물리적 딜레이**를 피할 수 없었습니다. 

한국경제나 네이버처럼 0초 만에 텍스트와 사진이 꽉 채워진 메인 화면을 보여주려면, 서버에서 HTML과 데이터를 미리 완벽하게 합쳐서(조리해서) 던져주는 **SSR(서버 사이드 렌더링)** 기술이 반드시 필요합니다.

## 💡 해결책: Next.js + Vercel
이 궁극의 성능을 구현하기 위해, 내일부터 **Next.js** 언어를 기반으로 Vercel이라는 전문 웹 호스팅 서버에 "본 서비스"를 구축합니다.

### Next.js 도입 시 3대 혜택
1. **진정한 0초 로딩속도 구현 (SSR 지원)**: 사용자가 클릭하는 순간 이미 기사와 사진이 데이터베이스와 합쳐져 구워진 완성본 화면이 뜨므로 빈 화면 깜빡임이 소멸합니다.
2. **검색엔진/카톡 공유(SEO) 폭발적 향상**: 서버에서 정보를 세팅해 주므로, 구글이나 네이버 로봇이 기사 내용을 완벽히 긁어가 검색 상단 노출에 최적화되며 카톡 공유 시 기사 전용 썸네일이 완벽히 뜹니다.
3. **미친 유지보수성 (Component)**: 헤더, 푸터 같은 파일들을 레고 블록처럼 분리하여 관리. 헤더 로고를 바꾸고 싶다면 딱 1개의 파일만 수정해도 사이트의 1만 개 페이지가 일괄 변경됩니다.

## 🛠 아키텍처 및 문법 변화 패러다임

### "HTML 안 쓰고 완전히 새로운 걸 배워야 하나요?"
👉 **아닙니다! HTML 지식을 99% 그대로 가져갑니다!**
* 기존처럼 HTML과 자바스크립트 파일을 나눌 필요 없이, 파일 한 곳(`.tsx` 또는 `.jsx`) 안에 섞어서 코딩하는 진화된 형태(**JSX**)를 다룹니다.
* `<div>`, `<img>`, `<a>` 등의 지식은 그대로 재활용하며, 이 JSX 코드는 방문자에게 도달할 때 Vercel 서버가 세상에서 가장 완벽한 **순정 HTML**로 알아서 번역해 줍니다.

## 🏃 내일 진행할 작업 로드맵 (Action Plan)

1. **Phase 1: 뼈대 공사 (Next.js 세팅)**
   - Vercel 전용 맞춤형 Next.js 신규 앱 초기화.
   - 기존의 Supabase 연결 키 및 환경 변수 안전 세팅.
2. **Phase 2: UI 블록 쪼개기**
   - 현재 잘 만들어 둔 `index.html` 기반의 바닐라 CSS/헤더/하단 메뉴를 똑 떼어서 React Component(부품)로 변환.
3. **Phase 3: 서버 사이드 연동 (SSR 적용)**
   - 메인 화면의 뉴스 띄우기 로직을 백엔드 단으로 옮겨, 브라우저가 버벅거리지 않고 바로 완성된 텍스트 덩어리를 받아보게 만듭니다.
4. **Phase 4: 자동 빌드 파이프라인 (CI/CD)**
   - 작업물을 깃허브에 올리기만 하면 Vercel이 알아서 서버를 재가동하여 서비스에 즉시 반영하는 24시간 무인 배포 시스템을 완성합니다.
