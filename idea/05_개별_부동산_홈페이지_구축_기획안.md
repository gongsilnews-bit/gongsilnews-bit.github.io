# 🚀 향후 개발 기획: 부동산 개별 맞춤형 홈페이지 제공 (White-label SaaS)

## 기획 배경 및 비즈니스 가치
- **중개사의 니즈:** 자신만의 번듯한 브랜드 홈페이지를 갖고 싶어 하나, 정작 본인이 보유한 매물 개수가 적어 빈약해 보이고 관리가 어려워 포기하는 경우가 많음.
- **공실뉴스의 강점(해결책):** 임대인 직거래 매물과 타 부동산의 '공동중개' 매물이 가득한 **거대한 전체 DB(Supabase)를 풀장(Pool)처럼 활용**. 
- **결과:** 중개사는 자신의 홈페이지를 만들자마자 수천 개의 매물이 꽉 찬 훌륭한 대형 포털급 사이트를 소유하게 됨. (쇼피파이 프로페셔널 버전이나 대형 프랜차이즈 홈페이지 시스템과 같은 원리)

## 기술적 구현 방안 비교 (DB는 공실뉴스 Supabase 전체 공유)

### 1안: 같은 호스팅/서버에서 구동 (멀티테넌시 방식, 추천 ⭐⭐⭐⭐⭐)
- **개념:** 기존 공실뉴스 호스팅(GitHub Pages 등) 하나만 사용하되, 주소창(예: `agent1.gongsilnews.com` 또는 `gongsilnews.com/agent1`)을 인식하여 화면의 로고, 테마 색상, 중개사 연락처만 동적으로 바꿔치기해서 보여주는 방식입니다.
- **장점:** 
  - **유지보수 극강:** 버그를 고치거나 새로운 기능을 추가하면 수백 명의 부동산 홈페이지가 '동시에' 1초 만에 업데이트됩니다.
  - **비용 절감:** 호스팅 서버 관리가 하나뿐이므로 운영비용이 거의 0원에 가깝게 유지됩니다.
- **단점:** 중개사별로 완전히 다른 기형적인 레이아웃(디자인 구조 자체를 엎는 것)을 요구할 때는 대응하기 까다롭습니다.

### 2안: 별도의 호스팅으로 소스 코드를 복제하여 배포 (독립형)
- **개념:** 중개사 A가 가입하면 새로운 호스팅 공간을 파서 소스코드를 통째로 복사해주고, DB 연결 주소만 공실뉴스 Supabase로 바라보게 하는 방식입니다. (`www.bestrealtor.com` 처럼 완전한 개인 도메인 설정 시 유용)
- **장점:** 중개사 입장에서 완전히 독립된 사이트를 가진 느낌을 주며, A 중개사만을 위한 특별 맞춤 기능이나 완전히 다른 디자인을 씌우기 편합니다.
- **단점:** **유지보수 지옥**이 열릴 수 있습니다. 사이트가 100개가 되면 매물 검색 기능에 버그가 생겼을 때 100개의 사이트를 일일이 업데이트하고 다시 배포해야 합니다.

## 종합 의견 및 제안 (Hybrid)
- 초기 서비스 시점에서는 무조건 **1안(같은 호스팅에서 로고/색상/연락처만 바뀌는 방식)**으로 가야 합니다. 관리가 생명이기 때문입니다.
- 나중에 중개사가 '자신만의 개인 도메인(.com, .co.kr)'을 붙이고 싶어 할 때를 대비해, Vercel이나 Cloudflare 같은 최신 호스팅의 **'커스텀 도메인 라우팅'** 기술을 사용하면, 코드는 하나(1안)로 유지하면서 겉보기에는 완벽한 개인 사이트(2안)처럼 보이게 만드는 하이브리드 기술을 적용하는 것이 수익화에 가장 베스트입니다.
